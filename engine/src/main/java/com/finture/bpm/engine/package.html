Public API of the Camunda Platform engine.

Typical usage of the API starts by the creation of a {@link com.finture.bpm.engine.ProcessEngineConfiguration} (typically based on a configuration file), from which a {@link com.finture.bpm.engine.ProcessEngine} can be obtained.

Through the services obtained from such a {@link com.finture.bpm.engine.ProcessEngine}, BPM and workflow operation can be executed:

{@link com.finture.bpm.engine.RepositoryService}: Manages {@link com.finture.bpm.engine.repository.Deployment}s
{@link com.finture.bpm.engine.RuntimeService}: For starting and searching {@link com.finture.bpm.engine.runtime.ProcessInstance}s
{@link com.finture.bpm.engine.TaskService}: Exposes operations to manage human (standalone) {@link com.finture.bpm.engine.task.Task}s, such as claiming, completing and assigning tasks
{@link com.finture.bpm.engine.IdentityService}: Used for managing {@link com.finture.bpm.engine.identity.User}s, {@link com.finture.bpm.engine.identity.Group}s and the relations between them
{@link com.finture.bpm.engine.ManagementService}: Exposes engine admin and maintenance operations, which have no relation to the runtime execution of business processes
{@link com.finture.bpm.engine.HistoryService}: Exposes information about ongoing and past process instances.
{@link com.finture.bpm.engine.FormService}: Access to form data and rendered forms for starting new process instances and completing tasks.
